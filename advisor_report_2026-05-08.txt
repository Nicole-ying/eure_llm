多智能体奖励函数自动设计框架 — 本周实验进展汇报

日期：2026-05-08

一、框架核心能力验证——LunarLander 六轮完整实验

本周我们获得了一组极具说服力的完整实验数据。在 LunarLander-v2 环境上
使用 1M steps/round × 6 轮迭代的完整 pipeline 运行结果显示，多智能体
框架具备了从零开始自主诊断奖励函数缺陷、逐步改进策略、最终学会任务
的核心能力。以下按轮次展示系统的自主进化过程。

实验条件：LunarLander-v2，PPO，16 envs GPU，每轮 1M steps，每轮约 60 分钟。

1.1 系统自主进化轨迹

| 轮次 | completion | fall | mean_length | 关键事件 |
|------|-----------|------|-------------|---------|
| R0   | 0%        | 100% | 63.3        | LLM 初始生成，智能体快速坠落 |
| R1   | 0%        | 100% | 58.2        | Analyst 诊断→调整惩罚系数，但幅度过大 |
| R2   | 0%        | 100% | 78.3        | 继续调参，存活时间开始延长 |
| R3   | 0%        | 100% | 514.2       | 存活能力大幅提升，但从不尝试着陆 |
| R4   | 90%       | 10%  | 400.1       | 突破！系统学会可靠着陆 |
| R5   | 80%       | 20%  | 797.1       | 飞行更稳定，但新增惩罚略微降低着陆率 |

关键的 Round 3 → 4 突破：系统自主诊断出稀疏终端奖励导致"学会着陆→
灾难性遗忘"的振荡，主动将策略从稀疏终端奖励切换为稠密逐帧塑形奖励
（per-step shaping），实现了从 0% 到 90% 的跳跃。这一诊断-决策过程
完全由多智能体系统自主完成，未依赖任何官方奖励函数的参考。

1.2 记忆系统的跨轮知识累积

Memory System 是本框架的核心区别性组件。在六轮迭代中，系统在 MEMORY.md
中自动积累的设计经验清晰展示了持续学习能力：

- R1 经验："outcome penalty 幅度是 per-step 奖励的 2000 倍，导致智能体
  学会快速失败" → 记录惩罚缩放原则
- R2 经验："惩罚仍过大，且缺乏着陆引导信号" → 记录奖励分量平衡原则
- R3 经验："稀疏终端奖励导致策略振荡" → 记录稠密塑形优于稀疏奖励的结论
- R4 经验："稠密塑形成功实现 90% 着陆" → 记录成功模式
- R5 经验："速度惩罚降低着陆率，智能体悬停不前" → 记录惩罚与奖励的权衡

每一轮的经验教训都被结构化为"变更→结果→原因→建议"的标准格式，自动
追加到 MEMORY.md 中供后续智能体读取。这种跨轮知识累积机制使得系统
能够在一个回合中"记住"上一个回合的失败模式，避免重复犯同样的错误。

二、记忆系统的跨轮知识累积（与 Eureka 的关键区别）

上述进化轨迹之所以可能，核心支撑是本框架独有的 Memory System。
Eureka（ICLR 2024）等现有方法每轮迭代之间完全无状态——每轮都是一次
独立的"从零开始"的优化尝试，无法从过去的失败中学习。我们的系统通过
在每轮迭代后自动维护两份文档实现跨轮知识累积：

- MEMORY.md：记录技术性记忆——奖励函数设计的因果经验。格式固定为
  "变更→结果→原因→建议"，每轮自动追加。如上文 R1-R5 的经验链所示，
  系统正是通过这些累积的记忆避免了反复犯同样的错误。
- TASK_MANIFEST.md：记录任务结构信息——观测空间、动作空间、环境
  step() 源码、终止条件等。这些信息由 Perception Agent 在 Round 0
  生成，并在后续轮次中供所有智能体参考，避免重复分析。

每轮 Reflection 阶段完成后，系统自动将迭代中获得的经验追加到记忆
文档中。这些记忆在下轮迭代中被 Perception Agent 和 Analyst Agent
读取，使系统能够在多轮迭代中积累设计知识，类似于人类的"做中学"过程。

值得一提的是，上述 R4 的关键突破（从稀疏奖励切换到稠密塑形）正是
建立在 R3 经验的基础之上——系统"记住"了稀疏终端奖励导致的策略振荡
问题，从而在 R4 中选择了完全不同的设计策略。


三、框架成熟度总结

当前 pipeline 已在两个环境上完整验证：
- 全自动运行：环境探索 → Prompt 生成 → LLM 调用 → PPO 训练 →
  多阶段评估 → Perception 分析 → Analyst 提案 → Generator 重写 →
  Reflection 评估 → Memory 更新（无需人工介入）
- 跨环境验证：LunarLander（GPU，16 envs，6 轮）和 MountainCar（CPU，
  4 envs，进行中），覆盖离散和连续动作空间
- 系统具备 resume 能力（从断点继续训练/评估/迭代）
- 单轮端到端耗时：约 34-60 分钟（取决于环境复杂度）
- 配置系统标准化：覆盖 BipedalWalker、LunarLander、MountainCar、
  CartPole、HalfCheetah 五个环境的超参数配置
