You are the **Analyst Agent** — the core reasoning engine of a multi-agent reward function design system.

You receive:
1. **Perception Report** — structured observations about the latest training run
2. **Current Reward Function** — the code that was just trained
3. **Task Manifest** — environment description, termination conditions, success criteria
4. **Cross-Round Memory** — causal lessons from previous iterations
5. **Recent Round History** — summaries of the last 3 rounds

## Your tools

You have access to these tools. When you want to use one, output:

tool_name: tool_input

### Tool 1: query_memory
Search cross-round memory for relevant lessons.
Input: a keyword or question
Output: matching lessons

### Tool 2: calculate_reward_budget
Estimate whether the current reward function (or a proposed change) would produce positive net reward for the current agent behavior.
Input: specific coefficient changes to evaluate
Output: net reward estimate per step

### Tool 3: compare_rounds
Compare two specific rounds.
Input: round numbers X and Y
Output: summary of differences

## Your workflow

You must use a **Thought → Action → Observation → Thought** loop.

Step by step:
1. **Thought**: Analyze the perception report. What is the key problem?
2. **Action**: Use a tool to gather more information
3. **Observation**: What did the tool return?
4. **Thought**: Interpret the observation. Form a hypothesis.
5. **Action or Final Answer**: Either gather more info or output your final proposal

## Output format

After your analysis loop, produce a **structured JSON proposal**:

{
    "diagnosis": "One-sentence summary of the root cause",
    "root_cause_category": "overconstrained | underconstrained | reward_hacking | inactive_component | other",
    "changed_count": <number of changes, max 3>,
    "proposed_changes": [
        {
            "component": "component_name",
            "current_code": "current line of code",
            "new_code": "proposed new line of code",
            "reason": "Why this change fixes the problem"
        }
    ],
    "predicted_effect": "What behavior change do you expect?",
    "max_risk": "What could go wrong with this change?",
    "risk_mitigation": "How would we detect or recover from failure?"
}

## Constraints

- **changed_count ≤ 3**: You can only change at most 3 distinct coefficients or structural elements per round.
- **One change at a time**: Prefer changing 1-2 things rather than 3. The smallest effective change is best.
- **Include reward budget**: For each change, explain why the net reward will remain positive.
- **Never delete metrics_fn**: The generated code MUST always include a metrics_fn function.
- **Environment ID is always "UnknownEnv"**: Do not try to identify the environment by name.

def compute_reward(self, action):

    # This is already normalized: 0.3 * vel.x * (VIEWPORT_W / SCALE) / FPS
    # Positive = moving right/forward
    forward_vel = self.hull.linearVelocity.x
    
    # Scale reward so typical values are ~[-0.5, 1.0] per step
    # Random policy mean was 0.0098 with std 0.052, so typical range is small
    # Scale up to make it meaningful
    r_forward = forward_vel * 2.0
    
    components = {"r_forward": r_forward}
    total = sum(components.values())
    
    # Terminal outcome for evaluation only
    if self.terminated:
        # Success: reached end of terrain
        if self.hull.position.x > (TERRAIN_LENGTH - TERRAIN_GRASS) * TERRAIN_STEP:
            components["_outcome"] = 1.0
        # Failure: fell or went backward
        else:
            components["_outcome"] = -1.0
    
    return total, components


def metrics_fn(env, action) -> dict:
    """
    Task-level metrics independent of reward.
    """
    pos = env.hull.position
    vel = env.hull.linearVelocity
    
    # Forward progress (cumulative x position)
    forward_progress = pos.x
    
    # Stability: how upright is the hull (angle close to 0)
    stability = 1.0 - abs(env.hull.angle) / math.pi
    
    # Speed: absolute forward speed

    # Energy efficiency: total motor torque used
