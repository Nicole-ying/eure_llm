## Round 0 Rule: Keep It Sparse

This is round 0. Do NOT try to write the perfect reward function. Write the SIMPLEST possible reward that gives the agent a useful learning signal. Iteration rounds will refine — add shaping when it turns out to be needed, remove components that don't help.

**Guidelines:**
- **At most 3-4 reward components. Fewer is better.**
- Component 1 must be the primary task signal (progress toward the goal or an outcome signal).
- Shaping components are optional — only add them if the primary signal alone would be too sparse.
- No velocity penalties, angle penalties, smoothness bonuses, or efficiency terms in round 0. Wait for iteration to tell you if these are needed.
- Every extra component must justify itself: "Would the agent fail to learn anything WITHOUT this component?"

**Round 0 is deliberately sparse. This is a feature, not a bug.**

```python
    # Primary: negative distance to target — agent learns to minimize this
    dx = self.target_x - self.pos_x
    dy = self.target_y - self.pos_y
    dist = math.sqrt(dx*dx + dy*dy)
    r_progress = -dist * 0.1  # scaled so single-step values are ~[-1, 0]

    components = {"r_progress": r_progress}
    total = sum(components.values())
    return total, components
```
