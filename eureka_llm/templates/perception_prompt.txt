## Instructions

Produce a structured markdown report with these sections:

### 1. Behavior Trend Summary
- At each evaluation milestone, what is the agent doing? (completing task? falling immediately? surviving but not moving?)
- Is the trajectory improving, regressing, or flat?
- Key numbers: mean_length and the most important env-specific metrics at final timestep

### 2. Critical Metrics
- List the 3 most important env-specific metrics and their trend
- Flag any metric that is moving in the wrong direction

### 3. Reward Component Health
- Which components are active (mean significantly non-zero)?
- Which components are dominant (|mean| > 2× others)?
- Which are inactive/negligible (mean ≈ 0 or std ≈ 0)?
- Is any component's mean value suspicious (e.g., exactly 0)?

### 4. Behavioral Diagnosis (1-2 sentences)
- In plain language, what is the agent's current strategy?
- Is it reward hacking? Stuck in a local optimum? Making genuine progress?

### 5. TDRQ Diagnosis
- Explain whether low TDRQ is mainly from imbalance, inactivity, or exploration collapse
- Give one sentence on whether this reward should be iterated, diversified (search/population), or kept

### 6. Key Numbers for Budget Calculation
Extract numerical metrics from the Evaluation Metrics table for use in reward budget analysis:
- mean_length
- All env-specific metrics listed in the Evaluation Metrics table (these are the primary task-performance indicators)

Output ONLY the report. No code. No proposals.
